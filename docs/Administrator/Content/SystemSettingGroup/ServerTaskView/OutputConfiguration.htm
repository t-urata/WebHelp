操作ログのファイル出力設定を行う

操作ログを外部ファイルへ出力する際の設定方法について説明します。

 

運用準備

操作ログを外部出力するには、[サーバタスクの設定]ビューでサーバタスク[操作ログのファイル出力]を有効に設定します。

 

操作ログの外部出力設定を行うには

操作ログの外部出力に関する各種設定は、以下のXMLファイルにて管理されています。

設定を変更するには、メモ帳などでファイルを直接開いて編集してください。

 

  • 　(TimeTracker FXインストールフォルダ)￥OperationLogServerTask.xml

 

  • 設定可能なパラメータ要素、および設定値には制限があります。

設定可能なパラメータ要素を追加したり、設定値を変更した場合には動作保証対象外となります。

    ディレクトリとファイル名

    filePath

    OperationLogs/{config_folder_name}/

    OperationLogServerTask.log

    • ディレクトリ名：絶対パス、または相対パスで指定します。

      • 絶対パス指定･･･指定されたパスのフォルダへ出力します。

      • 相対パス指定･･･インストールフォルダを基準として指定された相対パスのフォルダへ出力します。

      • 指定なし・・・インストールフォルダへ出力します。

      • パス内に{config_folder_name}を指定した場合、{config_folder_name}をサーバタスクが参照するサーバ設定ファイルが格納されているフォルダ名に置き換えます。

      サーバタスクが参照するサーバ設定ファイルの格納フォルダについては、こちらをご参照ください。

    • ファイル名：必須

    • 絶対パスとは・・・ファイルを格納するパス(場所)を最初から最後まで記述します。

    • 相対パスとは・・・基準となる場所から見て相対的なファイルの場所を記述します。

    ファイル名の編集方法

    staticLogFileName

    true

    • true：<filePath>要素にて指定されたファイル名です。

    • false：<filePath>要素にて指定されたファイル名の後に数字を追加します。

     

    文字コード

    encoding

    Shift_JIS

    以下の文字コードが指定できます。

    • Shift_JIS

    • EUC-JP

    • UTF-8

    		 

    書き込み方法

    appendToFile

    true

    • true：追加

    • false：上書き

    		  

    最大サイズ

    maximumFileSize

    10MB

    最大サイズと単位を指定します。

    ※以下の単位が指定できます。

    • KB

    • MB

    • GB

    最大サイズを超えるとログローテーションを行います。

    ファイル数の上限

    maxSizeRollBackups

    0

    • 0未満：バックアップファイルは削除しません。

    • 0：ログファイルの最大サイズ<maximumFileSize>になった時点で上書きします。

    • 1以上：バックアップファイル数。

    ログローテーションにより保持するファイル数の上限を指定します。

    ログローテーションの種類

    countDirection

    -1

    ログファイルの名前に付与する数値の順番を指定します。

    • -1より大きい値：ログファイルの作成時期が古い順に番号をつけます。

    • -1：ログファイルの作成時期が新しい順に番号をつけます。

    要素の組み合わせにより、ログファイルを以下のような運用で管理できます。

    • ログファイルを自動的には消さず、ログファイルの作成時期が古い順に番号を振って管理する。

    • <maxSizeRollBackups>要素を「0」未満に指定

    • <staticLogFileName>要素を「false」に指定

    • <countDirection>要素を「-1」より大きい値に指定

    • ログファイルを自動的には消さず、ログファイルの作成時期が新しい順に番号を振って管理する。

    • <maxSizeRollBackups>要素を「0」未満に指定

    • <staticLogFileName>要素を「true」に指定

    • <countDirection>要素を「-1」に指定

    • ログローテーションとはログファイルの肥大化を防ぐための仕組みです。
      指定された最大サイズを超えるとログファイルのバックアップを取り、新しいログファイルを作成します。
      バックアップファイルは指定されたログファイル数の中でローテーションされます。
